Color Usage Rules
✓ Use --accent for interactive elements, links, logo mark, key numbers
✓ Use --gold for "highest value" findings, warnings, important callouts
✓ Use --green for success states, "with Engram" columns, positive metrics
✓ Use --red for failure states, "without Engram" columns, negative metrics
✗ Never use purple gradients or white backgrounds — that's the "generic AI" aesthetic we avoid
✗ Never use full white text on dark — use --text-bright (#e6edf3) for contrast without harshness

Typography Rules
✓ H1 always gets padding: 8px 0 — this is the "longer height" fix. Never collapse it.
✓ H1 uses clamp() for fluid sizing — never fixed px on hero headlines
✓ Accent em spans in H1 for emphasis — never bold, never underline
✓ All body text uses IBM Plex Mono — maintains terminal/technical character throughout
✗ Never use Inter, Roboto, or system-ui — these are generic and undermine the aesthetic
✗ Never use font-size below 10px or above 64px in content areas

05 — Texture & Atmosphere
Noise Overlay
✓ Always include the SVG noise overlay on body::before — opacity 0.4. It prevents the flat-dark-background look.
✓ Terminal blocks use #060910 (slightly different from --bg) for subtle depth.
✓ Use box-shadow: 0 0 16px var(--accent) on the logo mark inner element — the only glow effect used.
✗ No gradients on backgrounds — only on card overlays using rgba at 0.03–0.05 opacity.
✗ No drop shadows on text — only on interactive elements.
